# Voyt Ruslan Task 2: Deconvolution

В этой папке находится итоговое решение задачи деконволюции изображения и ноутбуки с экспериментами по подбору параметров.

## Состав

- [solution.py](solution.py) — финальное решение.
- [exp.ipynb](exp.ipynb) — ранние эксперименты.
- [second_exp.ipynb](second_exp.ipynb) — основной ноутбук с подбором параметров.
- [task_testdata](task_testdata) — пример из задания.
- [generated_testdata](generated_testdata) — синтетические тестовые случаи для отладки.

## Что делает решение

Алгоритм восстанавливает изображение из размытого входа по известному ядру свёртки и заданному уровню шума.

Используется TV-регуляризация и primal-dual итерационная схема с:

- зависимостью `lambda_tv` от `noise_level`,
- фиксированным числом итераций `100`,
- параметрами схемы `tau = 1.2`, `sigma = 0.05`, `theta = 1.0`,
- ускоренной свёрткой через `cv2.filter2D`.

Для режима с малыми параметрами зашумления (`noise_level <= 2`) дополнительно слегка ослабляется `lambda_tv`.

## Как запустить

Формат запуска соответствует условию задачи:

```powershell
python solution.py <input_image> <kernel> <output_image> <noise_level>
```

Пример:

```powershell
python solution.py task_testdata\blurred.bmp task_testdata\kernel.bmp task_testdata\result.bmp 1.289
```

## Пример на кейсе

Для `task_testdata` при оценённом уровне шума `~1.289` решение даёт:

- `input PSNR = 28.4200 dB`
- `output PSNR = 32.9488 dB`
- `delta PSNR = +4.5288 dB`

Это означает, что восстановленное изображение заметно ближе к `reference.bmp`, чем исходное размытое.

## Зачем нужны ноутбуки

Ноутбуки использовались для:

- исследования зависимости `lambda_tv` от уровня шума,
- подбора параметров `tau` и `sigma`,
- проверки разных конфигураций на тестовых изображениях.
